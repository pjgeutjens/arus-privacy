Arus Privacy Policy

Arus is a local Android app provided by Leap Consulting for tracking selected bank transaction notifications. Arus is designed to keep financial data on the user's device unless the user explicitly chooses to export or share it.

Effective date: July 9, 2026

Arus does not run a cloud service, does not sell personal data, and does not automatically upload transaction, balance, account, or notification data to Leap Consulting or any third party.

Who Provides Arus

Arus is provided by Leap Consulting. For privacy questions, use the developer contact email address published for Arus on Google Play.

What Arus Does

Arus helps the user create a local transaction timeline and estimated account balances from bank app notifications. The app currently targets supported bank notification formats for Jenius, Livin' by Mandiri, and CIMB OCTO. Supported banks and formats may change over time as bank apps change their notifications.

Notification Access

Arus can access bank notification content only after the user enables Android notification access for Arus. Once enabled, Android may deliver supported bank notifications to Arus even when the Arus app is closed or not actively being used. Arus uses this access to detect transaction notifications, parse transaction details, and store a local event log for the user's transaction history.

The user can disable notification access at any time in Android system settings. If notification access is disabled, Arus will stop receiving new notifications.

Data Arus Accesses Or Stores

Depending on the user's setup and what a bank includes in its notifications, Arus may access, process, or store the following data on the device:

Data type Examples Purpose
Bank notification data Bank app name, Android package name, notification title, notification text, expanded notification text, posted time, captured time, and local notification hash. Detect supported bank notifications, parse transactions, diagnose unsupported notification formats, and avoid duplicate processing.
Financial and payment information Transaction amount, currency, money in or money out direction, merchant, sender, recipient, description, transaction timestamp, bank, account label, and parser status. Create a local transaction ledger and calculate balances from user-confirmed checkpoints.
User-entered account data Bank selection, account alias, notes, manually entered balance checkpoints, and balance confirmation timestamps. Let the user organize transactions by account and maintain estimated per-account and global balances.
Diagnostics data Parser version, app version, supported bank package names, raw notification records, parse status, transactions, accounts, and replay health. Help the user or developer understand whether bank notification formats are being parsed correctly.

Data Arus Does Not Intentionally Access

  • Arus does not request location, contacts, microphone, camera, SMS, phone call logs, or calendar access.
  • Arus does not ask for bank login credentials, PINs, passwords, one-time passwords, or card numbers.
  • Arus does not intentionally process non-bank notifications. Unsupported app notifications are ignored by the app's allowlist.

How Data Is Used

Arus uses captured notification data to build a local transaction timeline, estimate balances from user-confirmed balance checkpoints, detect unsupported notification formats, export CSV files when requested by the user, and create user-requested diagnostics reports.

Data Sharing

Arus does not automatically transmit personal or sensitive user data to Leap Consulting, cloud servers, analytics providers, advertising networks, or data brokers. Arus does not sell personal or sensitive user data.

The user may choose to export CSV files or diagnostics reports. Those exports are user-initiated. When the user chooses to share an export through email, messaging, cloud storage, or another app, the chosen app or service handles the shared file under its own privacy practices.

Storage And Security

Arus stores app data locally on the Android device. The local database is encrypted at rest using SQLCipher, with database access controlled by the Android app sandbox and the app's local encryption key handling. Arus does not operate a remote database or synchronization server.

Exports created by the user may be stored temporarily in app cache or in the destination selected by the user. Exported files are no longer controlled by Arus after the user shares or stores them outside the app.

Diagnostics And Sharing

Arus can create a diagnostics report to help improve notification parsing. This report may include raw notification text, parsed transaction data, account aliases, balance checkpoints, parser status, and app metadata. The report is created only when the user taps the diagnostics export action and chooses where to send it through Android's share or email interface.

Arus does not automatically send diagnostics reports. The user controls whether a report is shared, which app is used to share it, and who receives it.

Third Parties

Arus does not include advertising SDKs, analytics SDKs, or third-party tracking SDKs. Arus uses Android platform features such as notification listener access, local storage, file sharing, and the Android share sheet.

Data Retention And Deletion

Arus keeps local notification, transaction, account, and balance data until the user deletes it, wipes data inside the app, uninstalls the app, or clears app storage through Android settings. Deleting local app data removes the local database from the device. Files that the user previously exported or shared must be deleted separately from the place where the user saved or sent them.

User Controls

  • The user can disable notification access in Android settings at any time.
  • The user can delete local app data using the app's wipe action or Android system storage controls.
  • The user can choose whether to export diagnostics or CSV files.
  • The user can choose whether to set up accounts and balance checkpoints. Arus can capture supported bank notifications for analysis even before accounts are configured.

Legal Requests And Safety

Because Arus does not operate a cloud service for transaction data, Leap Consulting generally does not possess users' local transaction records. If Leap Consulting receives a valid legal request for information it actually holds, it will respond according to applicable law.

Children

Arus is not intended for children and is not designed to collect data from children.

Changes

This policy may be updated when Arus changes how it processes, protects, or shares data. The effective date above will be updated when material changes are made.
